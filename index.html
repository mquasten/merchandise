Merchandise

Storedemo Webapplication with Spring, JPA, Primefaces

Download this project as a .zip file Download this project as a tar.gz file

Artefacts:

merchandise.domain

The domain objects that represents the model. The aggregate roots are:

  • Customer (Persons, Contacts)
  • CommercialSubjects (the abstraction from products and services, without commercial conditions)
  • Opportunities (CommercialSubjects enhanced with commercial conditions, abstraction from tender, tender offer, Product and ServicePackages)

merchandise.application

The application Layer, providing services for merchandise.web and merchandise.rest For each aggreate root a service is implemented to use the domain object. The repositories implementations are in application at the moment. Later it can be extracted into merchandise.infra. Repository-Implementations are done for JPA and in special stories for solr full text search. An AuthentificationService is available for Security. This is done using Spring Security (for pages and methodbased security on the services. The servicelayer is implemented without any remoting. Remoting can be done later using rest based services in merchandise.rest. Merchandise.rest can use merchandise. application and can provide the services simply using spring mvc controllers with annotation.

merchandise.web

A primefaces based web fronntend for the stories

  • as an interested new user, I will be able to register as a legal or natural person that represents a customer or as user for an existing customer, to be able to work with the System. I want to be able to login to the system. I want that only authorized users can offer products, give bids to my tenders.
  • As a provider from products and services or as a tender I want to able to create CommercialSubjects, Opportunities and CommmercialConditions.
  • As a biider or customer that will by simply products i want to be able to find products, tenders. I want to be able to buy then or to create a bid for them.
  • As a tenderer or a provider from products I will be able to see who has orderd my Products and services or which bids exists for my tender. I want to confirm the order or to give the sale by tender.
